## 字符
字符类型char会以**32位**值表示单个**Unicode**字符。

Rust会对单独的字符使用char类型，但对字符串使用UTF-8编码。因此，**String会将其文本表示为UTF-8字节序列，而不是字符数组**。

UTF-8字节序列会比字符数组省空间，因为有些字符在UTF-8字节序列中只占用一个字节，而每个char类型都是4字节。

## 元组

元组的每个元素可以有不同的类型：

```rust
fn main() {
    let value: (&str, bool, i32) = ("test", false, 100);
    
    println!("{}", value.0);  // test
    println!("{}", value.1);  // false
    println!("{}", value.2);  // 100
}
```

### 零元组 ()

此类型只有一个值，写作 ()。当无法携带任何有意义的值但其上下文仍然要求传入某种类型时，Rust就会使用`()`。

不返回值的函数的返回类型为`()`。

## Option<T> & Some(V)

```rust
fn main() {
    let num: u8 = 100;

    let result1: Option<u8> = num.checked_add(200);
    println!("{:?}", result1);  // None
    // let result_num: u8 = result1.expect("extract num error"); 
    // 这里提取数据会panic，并打印出extract num error信息

    let result2: Option<u8> = num.checked_add(10);
    println!("{:?}", result2);  // Some(110)

    // 提取出数字
    let result_num: u8 = result2.expect("extract num error");
    println!("{}", result_num);  // 110
}
```

`checked_add` 是 Rust 为所有整数类型提供的**安全加法方法**，作用是：

1. 尝试将调用者的值与参数相加；
2. 如果相加结果**超出该类型的取值范围**（溢出），返回 `None`；
3. 如果结果有效，返回 `Some(结果值)`。

`Option<T>` 是 Rust 的核心枚举类型，定义如下：

```rust
enum Option<T> {
    None,    // 表示“无值”（加法溢出）
    Some(T), // 表示“有值”（加法成功，T 是具体值的类型）
}
```

它强制调用者处理 “失败场景”（比如 `None`），避免空指针或未处理的错误，这也是 Rust 无空指针的关键原因之一。
